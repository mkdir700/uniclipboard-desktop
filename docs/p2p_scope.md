P2P Scope (LAN Discovery + Pairing)

本文件定义本迭代的 P2P 范围（局域网发现 + 连接 + 配对持久化）。
目标是明确边界，防止功能扩散，确保每一步都可运行、可验证、可回滚。

## Goals / 目标

- 在局域网内完成设备发现、连接与显式配对。
- 保障配对流程可控、可审计（必须显式确认）。
- 为后续扩展（DHT、Relay、Hole Punching）预留清晰的边界。

## Non-Goals / 非目标

- 不做全网 DHT。
- 不做 Relay 或 Hole Punching。
- 不做跨 NAT 的复杂穿透。
- 不做“发现即信任”的自动配对。

## Scope Constraints / 范围约束

- **仅做 LAN**：mDNS 为主的局域网发现。
- **显式配对**：任何新设备配对必须用户确认。
- **最小可落地**：每一步可单独验证和回滚。

## Validation / 验收

- 在同一局域网内可发现对端设备（mDNS）。
- 可建立连接并触发配对流程。
- 配对必须经过显式确认才能完成。

## Rollback / 回滚

- 任意步骤失败可回退至“仅本地运行、无 P2P 连接/配对”。
- 回滚不影响本地设备身份与数据存储。
